# AGENTS.md

Правила для агентов и разработчиков, работающих с этим репозиторием. Подробное
описание архитектуры — в [README.md](README.md).

## Архитектура (кратко)

Браузерное расширение (Chrome, Manifest V3) для списания времени в YouTrack. Код
исполняется в двух контекстах, и от этого зависит раскладка файлов:

- `popup/` — UI на Vue 3, никогда не видит токен.
- `background/` — service worker, единственный владелец токена.
- `shared/` — общий контракт (типы сообщений, DTO, сериализация `Either`).
- `storage/` — типизированная обёртка над `chrome.storage`.

Попап общается с воркером через `chrome.runtime.sendMessage`; функции API
возвращают наружу `Either<KyError, T>` вместо исключений.

## 1. Требования ИБ (обязательно)

Изоляция токена — главное требование, см. [README.md](README.md) строки 55-64.

- Пользовательский токен читается и используется ТОЛЬКО в service worker.
- Попап не хранит токен, не подставляет его в заголовки, не светит в URL/логах.
- Все запросы к YouTrack идут через воркер; `Authorization: Bearer` ставится
  внутри воркера.
- `popup/` не импортирует `background/*` статически — только динамический
  `import()` под `import.meta.env.DEV`, иначе токен и `ky` попадут в бандл
  попапа.
- Границы контекстов держим в структуре папок: `background/ ↔ popup/ ↔ shared/`.

## 2. Обновление CHANGELOG

После любых изменений обновляй [CHANGELOG.md](CHANGELOG.md): добавь запись в
раздел `[Unreleased]` (Добавлено / Изменено / Исправлено) — кратко и по сути.

## 3. Комментарии в коде

По умолчанию комментарии в коде не оставляем — код должен быть
самодокументируемым (понятные имена, небольшие функции).

- Допустимо: пояснение НЕтривиального функционала — неочевидные инварианты,
  ограничения платформы, компромиссы, ответ на вопрос «почему», а не «что».
- Недопустимо: комментарии, дублирующие код («// увеличиваем счётчик»),
  заголовки-разделители, закомментированный код, очевидные docstring'и.
